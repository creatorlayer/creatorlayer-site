How it works

Creatorlayer connects to platforms like YouTube and Twitch via their official OAuth flows. Lenders direct creators to a secure authorization link, and Creatorlayer returns a normalized verification payload.

  • Lender sends the creator a secure Creatorlayer OAuth link.
  • Creator authorizes access to their revenue and performance data on supported platforms.
  • Creatorlayer aggregates income, volatility and risk markers and returns a structured API response.

Data, consent & compliance

Creatorlayer only accesses creator data after an explicit OAuth authorization on the underlying platform (for example “Sign in with YouTube” or “Connect Twitch account”). We do not scrape, store passwords or bypass platform protections.

  • Explicit creator consent via each platform’s OAuth flow.
  • Purpose limitation: income verification, risk analysis and portfolio monitoring.
  • EU‑native hosting to support GDPR compliance.
  • Creatorlayer acts as infrastructure; lenders remain responsible for credit decisions and remain data controllers for their portfolios.

We can provide technical and data‑processing notes that your DPO can attach to internal records of processing activities.

Technical overview

Creatorlayer exposes a REST API that aggregates creator income signals from platforms such as YouTube and Twitch into a normalized verification object that fits existing credit engines and back‑office tools.

  • Backend: Node.js / Express with PostgreSQL.
  • Access pattern: lender creates a verification request, then sends the creator to an OAuth link.
  • Outputs: 30‑day and 90‑day income, trends, simple volatility markers, basic fraud flags, suggested advance / limit.
  • Integration: server‑to‑server JSON API; batch exports and ABS‑style tapes on the roadmap.
